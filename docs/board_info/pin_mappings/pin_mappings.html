Pin Mappings

Overview

The Bluejay breaks out 39 of the iCE40's PIO pins, which are connected to header pins D0–D38. When programming the iCE40, the top-level inputs and outputs of the RTL design must be assigned to physical pins on the FPGA package. This information is specified in a pin constraints file (.pcf) and is provided to the place-and-route tool as part of the bitstream generation flow.

When creating a .pcf file, it is important to know how the iCE40 pins are connected to the board's header pins. This mapping is provided in the table below.

The specific chip used on the Bluejay is the iCE40LP8K-CM81, which comes in an 81-ball ucBGA package. The balls are laid out in a 9x9 grid array, with each ball designated by a row letter (A through J, skipping I) and a column number.

Note that the connections between the MOSI and MISO pins and the iCE40 depend on the configuration mode. In Master Mode, MOSI is connected to G6 (IOB_105_SDO) and MISO is connected to H7 (IOB_106_SDI). In Slave Mode, these connections are reversed.

Oscillator Pin Mapping

In addition to the header pin connections listed in the table above, one other pin mapping is of particular relevance. This is the connection between the iCE40 and the board's 12 MHz oscillator, which provides a clock signal for the FPGA. The oscillator's 12 MHz square wave output is connected to ball C4 (IOT_198_GBIN0) of the FPGA.
