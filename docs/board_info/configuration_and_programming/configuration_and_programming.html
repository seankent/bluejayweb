Configuration and Programming

Overview

The iCE40LP8K is an SRAM-based FPGA, which means its configuration is volatile and must be reloaded each time the device is powered on.

The iCE40's configuration is loaded through an SPI interface. This process can either be controlled by the iCE40 itself or by an external programmer. Lattice refers to these two operating modes as Master Configuration Mode and Slave Configuration Mode respectively. The Bluejay supports both modes (refered to as Master Mode and Slave Mode for short).

The following sections explain how to set up and use both configuration modes, as well as how to program the flash chip for persistent storage of your bitstream.

Configuration Modes

Master Mode

In Master Mode, the FPGA automatically configures itself from the flash chip when power is applied or CRESET_B is de-asserted. In this mode, the FPGA acts as a SPI master and reads the configuration from the flash chip. By storing the FPGA's configuration in onboard non-volatile memory, the FPGA can configure itself and operate without being connected to an external programmer.

To operate the board in Master Mode, the configuration jumper should be placed in the MM position.

Programming the Flash

Before the FPGA can operate in Master Mode, a bitstream must be written into the board's flash memory chip. This can be done using an external programmer attached to the board's configuration interface.

The board's flash chip has a 2 Mbit capacity. The size of the bitstream image for an iCE40LP8K is 1,081,464 bits.

When programming the flash, the configuration jumper must be placed in the MM position.

Slave Mode

In Slave Mode, an external programmer loads the configuration directly onto the FPGA. This mode allows the FPGA to be quickly programmed and reprogrammed, which is useful when prototyping and debugging. Slave Mode can also be useful when the device is integrated into a larger system where an application processor can store and load the configuration.

To operate the board in Slave Mode, the configuration jumper should be placed in the SM position.

Configuration Jumper

The board includes a 3-pin header for selecting the configuration mode. To select Master Mode (MM), the jumper should be set to the MM position. To select Slave Mode (SM), the jumper should be set to the SM position.

The jumper must be in either the MM or SM position. It cannot be left disconnected.

Configuration Interface

The FPGA's bitstream is loaded through a SPI interface. In addition, the FPGA has two dedicated configuration pins, CRESET_B and CDONE, which assist in the configuration process. CRESET_B is an active-low input that resets the FPGA's configuration. CDONE is an output of the FPGA and is asserted when the configuration has been successfully loaded.

The board's configuration interface is located at the bottom of the J1 header and is shown in the figure below. This interface includes the four SPI signals as well as the two FPGA-specific configuration pins, CRESET_B and CDONE. These are abbreviated CRST and CDN on the board. The configuration interface can be used to program both the FPGA and the flash chip.

When connecting an external programmer to the board, both devices should operate off the same 3.3V supply. This can be achieved by powering the Bluejay from the external programmer's power supply through the 3V3 and GND pins.

Additional Information

The following sections provide more detailed information about the Master Mode and Slave Mode setup and sequences. While not strictly necessary for basic usage, they may be helpful for understanding how the configuration process works.

Configuration Circuit Overview

The SPI connections required on the board differ depending on the operation being performed. In Master Mode, the FPGA is the SPI master and the flash chip is the SPI slave. In Slave Mode, the external programmer is the SPI master and the FPGA is the SPI slave. When programming the flash, the external programmer is the SPI master and the flash chip is the SPI slave.

To support both configuration modes (and the flash programming), the Bluejay uses a set of analog switches controlled by the configuration mode jumper. These switches modify the SPI connections as needed.

The Bluejay's configuration mode jumper has two valid settings: MM and SM. These two settings are sufficient for supporting the three operations mentioned above (Master Mode, Slave Mode, and flash programming).

Master Mode Circuit

The MM setting is used for operating the FPGA in Master Mode and for programming the flash chip. The diagram below shows the SPI connections when the jumper is in the MM position.

In this configuration, the SPI_SS, SPI_SCK, SPI_SO, and SPI_SI pins of the FPGA are connected to the CS, SCK, SI, and SO pins of the flash. This establishes the SPI connections to allow the FPGA to send commands and read data from the flash.

The configuration interface pins also connect to the CS, SCK, MOSI, and MISO pins of the flash chip. This enables an external programmer to access and program the flash.

When programming the flash chip, the CRESET_B pin of the FPGA must be held low. This puts the FPGA into reset and prevents it from interfering with the flash programming sequence. It also places the FPGA's SPI pins in a high-impedance state.

Master Mode Sequence

When power is applied to the board, the CRESET_B pin of the FPGA is pulled high by a 10 kΩ pull-up resistor. This brings the FPGA out of reset. The FPGA then samples the value of its SPI_SS pin. With no external programmer connected, the SPI_SS pin is also pulled high by a 10 kΩ pull-up resistor. The FPGA therefore reads a logic high and enters Master Mode. The FPGA then begins the auto-configuration sequence. It starts by sending a wakeup command to the flash, then begins reading the bitstream starting at address 0x0. Once all the configuration data has been loaded, the FPGA sends a "deep power-down" command to the flash to put it into a low-power state. Finally, the FPGA drives its CDONE pin high to indicate successful configuration.

For more details on Master Mode, refer to the "iCE40 Programming and Configuration" manual.

Flash Programming Sequence

To program the flash, the external programmer must first put the FPGA into reset by asserting CRESET_B. It can then erase, program, and verify the flash's contents using standard SPI flash commands. When flash programming is complete, the external programmer can release CRESET_B. However, it is important that any SPI outputs of the external programmer are first placed into a high-impedance state or disconnected from the board. In general, this is necessary whenever an external programmer is attached to the board and CRESET_B is de-asserted.

Once CRESET_B is de-asserted, the FPGA will enter Master Mode and read in the new bitstream from the flash.

For more details on programming the flash chip, refer to the AT25DF021 datasheet.

Slave Mode Circuit

The SM setting is used for operating the FPGA in Slave Mode. The diagram below shows the SPI connections when the jumper is in the SM position.

In this configuration, the board's MOSI and MISO pins are connected to the FPGA's SPI_SI and SPI_SO pins. This allows an external programmer to write data into the FPGA. In addition, the CS pin of the flash chip is disconnected from the board's CS pin and pulled high by a 10 kΩ pull-up resistor. This keeps the flash in a de-selected state and prevents the signaling between the external programmer and the FPGA from accidentally modifying its contents.

Slave Mode Sequence

To put the FPGA into Slave Mode, the external programmer first resets the FPGA by driving the CRESET_B pin low. It also drives the SPI_SS pin low. The external programmer then releases CRESET_B while keeping SPI_SS held low. When the FPGA comes out of reset, it samples the SPI_SS pin. Upon reading a logic low, the FPGA enters Slave Mode. The external programmer can then load the bitstream onto the device. Once the configuration has been loaded, the FPGA drives its CDONE pin high to indicate the process has completed.

For more details on Slave Mode, refer to the "iCE40 Programming and Configuration" manual.
