Programming the Board

Overview

With your bitstream generated, it's time to load the design onto the FPGA. This page covers how to connect a programmer to the Bluejay and load your bitstream.

What You'll Need

You'll need an FTDI-based USB programmer to connect your computer to the board's configuration interface. Two common options are:

  • FTDI C232HM USB-to-serial cable
  • Adafruit FT232H breakout board

Both use the FT232H chip to perform USB-to-SPI conversion, allowing you to program the FPGA.

Understanding the Configuration Interface

Bitstreams are loaded onto the iCE40 using a SPI interface (CS, SCK, MOSI, MISO) and two dedicated configuration signals:

  • CRESET_B (abbreviated CRST) - Configuration reset
  • CDONE (abbreviated CDN) - Configuration done

The Bluejay's CONFIG interface (located at the bottom of header J1) provides access to these signals, along with power and ground.

Programmer Setup

FTDI C232HM USB-to-Serial Cable

The FTDI C232HM cable has a USB Type-A connector on one end and 10 color-coded wires with female headers on the other.

Cable Pinout:

Pin Pin Name Pin Color Type Description
1 VCC Red Output 3.3V power output
2 SK Orange Output Serial clock
3 DO Yellow Output Serial data output
4 DI Green Input Serial data input
5 CS Brown Output Serial chip select
6 GPIOL0 Gray Input/Output General purpose input/output
7 GPIOL1 Purple Input/Output General purpose input/output
8 GPIOL2 White Input/Output General purpose input/output
9 GPIOL3 Blue Input/Output General purpose input/output
10 GND Black GND Ground

Connections to Bluejay:

Bluejay Pin Name FTDI Cable Pin Name
3V3 VCC
GND GND
CS CS
SCK SK
MOSI DO
MISO DI
CRST GPIOL3
CDN GPIOL2

The cables are not labeled, so connections must be made based on wire color.

Adafruit FT232H Breakout Board

The Adafruit FT232H breakout board connects to your computer via USB-C and breaks out the FT232H I/O on two 11-pin headers.

Connections to Bluejay:

Bluejay Pin Name FT232H Breakout Board Pin Name
3V3 3V
GND GND
CS D4
SCK D0
MOSI D1
MISO D2
CDN D6
CRST D7

Programming the FPGA (Volatile)

This method loads the bitstream directly into the FPGA's SRAM. The configuration is lost when power is removed.

Board Setup

  1. Set the configuration mode jumper to SM (Slave Mode)
  2. Connect the LED EN jumper to enable the LEDs
  3. Disconnect USB and 5V power - the programmer will supply 3.3V through the 3V3 and GND pins

Programming Command

Once the programmer is connected, run:

Terminal
sudo iceprog -d i:0x0403:0x6014 -S gen/blinky.bin

The -S flag tells IceProg to program the FPGA's SRAM directly.

If successful, the USR LED should start blinking at 1 Hz!

Using the Makefile

If you created the Makefile in the previous step, you can program the FPGA with:

Terminal
make prog

Programming the Flash (Optional)

To make your design persistent across power cycles, you can program the onboard flash chip. The FPGA will then automatically configure itself from flash on each power-up.

Board Setup

  1. Set the configuration mode jumper to MM (Master Mode)
  2. Keep all other connections the same

Programming Command

Run:

Terminal
sudo iceprog -d i:0x0403:0x6014 gen/blinky.bin

Note the absence of the -S flag. Without it, IceProg defaults to programming the flash chip.

After programming completes, IceProg releases the FTDI pins, allowing CS and CRST to be pulled high. This prompts the FPGA to enter Master Mode and load the bitstream from flash. The USR LED should start blinking immediately.

You can now disconnect the programmer. The design will automatically load whenever the board is powered on.

Using the Makefile

If using the Makefile:

Terminal
make flash

Congratulations!

You've successfully created and programmed your first FPGA design on the Bluejay!
