Building the Bitstream

Overview

With your design files created, it is now time to generate the bitstream. This process involves three steps: synthesis, place-and-route, and bitstream packing.

Workspace Setup

To keep your workspace organized, create a gen/ subdirectory to store all generated files:

mkdir gen/

This keeps the design files separate from the outputs created by the bitstream tools.

Synthesis

The first step is logic synthesis, which converts the high-level RTL into a netlist of FPGA primitives.

Run yosys with the following command:

yosys -p "synth_ice40 -top blinky -json gen/blinky.json" blinky.sv

The -top blinky option specifies that the top module is named blinky. The -json gen/blinky.json option tells the tool to write the synthesized netlist to a JSON file of the given name. The source file containing the RTL design, blinky.sv, is provided as the final argument. Note, if there are multiple source files, add each as a space separated list.

yosys will output information about the synthesis process and report the resources used.

Place and Route

The next step is place-and-route, which maps the netlist to physical resources on the FPGA.

Run nextpnr-ice40 with the following command:

nextpnr-ice40 --lp8k --package cm81 --json gen/blinky.json --pcf blinky.pcf --asc gen/blinky.asc

The --lp8k and --package cm81 options specify the target FPGA (the iCE40LP8K-CM81). The --json gen/blinky.json option points the tool to the synthesized netlist. The --pcf blinky.pcf option points the tool to the pin constraints file. Finally, the --asc gen/blinky.asc option specifies where to write the placed-and-routed design.

nextpnr-ice40 will report timing information and confirm successful place-and-route.

Understanding the Timing Report

During place-and-route, nextpnr-ice40 uses the set_frequency constraint from the pin constraints file to perform timing analysis. This constraint tells the tool that the clk signal runs at 12.0 MHz. The tool will analyze whether the design can meet timing at this frequency and report the maximum achievable frequency (Fmax).

You should see the following line printed in the output:

Info: Max frequency for clock 'clk$SB_IO_IN_$glb_clk': 118.30 MHz (PASS at 12.00 MHz)

This confirms that the design meets timing with significant margin. If timing fails for whatever reason (e.g., if you set the clock frequency to 200 MHz), you will see something like this:

ERROR: Max frequency for clock 'clk$SB_IO_IN_$glb_clk': 118.30 MHz (FAIL at 200.00 MHz)

Bitstream Packing

The final step is to convert the placed design into a binary bitstream.

Run icepack with the following command:

icepack gen/blinky.asc gen/blinky.bin

This creates gen/blinky.bin, which contains 1,081,464 bits of configuration data for the iCE40LP8K.

Project Structure

After running the synthesis, place-and-route, and bitstream packing commands, your project directory should look like this:

.
├── blinky.pcf
├── blinky.sv
└── gen
   ├── blinky.asc
   ├── blinky.bin
   └── blinky.json

The gen/ directory now contains the intermediate and final output files from the bitstream generation process: - blinky.json - Synthesized netlist from Yosys - blinky.asc - Placed-and-routed design from NextPNR - blinky.bin - Final bitstream ready for programming

Using a Makefile (Optional)

For larger projects or when iterating on designs, manually running these commands can become tedious. A Makefile automates the build process.

Create a file called Makefile in your project directory:

# Project configuration
PROJECT = blinky
TOP_MODULE = blinky
SOURCES = blinky.sv
PCF = blinky.pcf

# Output directory
BUILD_DIR = gen

# iCE40 device configuration
DEVICE = lp8k
PACKAGE = cm81

# Tool configuration
YOSYS = yosys
NEXTPNR = nextpnr-ice40
ICEPACK = icepack
ICEPROG = iceprog

# Build targets
JSON = $(BUILD_DIR)/$(PROJECT).json
ASC = $(BUILD_DIR)/$(PROJECT).asc
BIN = $(BUILD_DIR)/$(PROJECT).bin

# Default target
all: $(BIN)

# Create build directory
$(BUILD_DIR):
    mkdir -p $(BUILD_DIR)

# Synthesis
$(JSON): $(SOURCES) | $(BUILD_DIR)
    $(YOSYS) -p "synth_ice40 -top $(TOP_MODULE) -json $@" $(SOURCES)

# Place and route
$(ASC): $(JSON) $(PCF)
    $(NEXTPNR) --$(DEVICE) --package $(PACKAGE) --json $(JSON) --pcf $(PCF) --asc $@

# Generate bitstream
$(BIN): $(ASC)
    $(ICEPACK) $(ASC) $@

# Program FPGA (volatile)
prog: $(BIN)
    sudo $(ICEPROG) -d i:0x0403:0x6014 -S $(BIN)

# Program flash (persistent)
flash: $(BIN)
    sudo $(ICEPROG) -d i:0x0403:0x6014 $(BIN)

# Clean build artifacts
clean:
    rm -rf $(BUILD_DIR)

# Phony targets
.PHONY: all prog flash clean

With the Makefile in place, you can build and program your design with these commands:

Build the bitstream:

make

Program the FPGA (SM mode):

make prog

Program the flash (MM mode):

make flash

Clean build artifacts:

make clean

The Makefile automatically handles dependencies, so if you modify your source files and run make, it will rebuild only what's necessary.
