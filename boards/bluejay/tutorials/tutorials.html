Getting Started

Overview

This guide walks you through creating your first FPGA design for the Bluejay and programming it onto the board. We'll build a simple LED blinker to demonstrate the complete workflow from RTL to bitstream.

In this tutorial, you'll create a simple module that blinks the board's USR LED on and off at 1 Hz.

Requirements

Before starting, make sure you've installed the required tools by following the Software Installation guide.

Blinky RTL

We'll start by creating a SystemVerilog module named blinky and save it to a file called blinky.sv.

The module has one input, clk, which connects to the board's 12 MHz oscillator, and one output, out, which connects to the USR LED.

Systemverilog
module blinky
#(parameter MAX_COUNT = 6000000)
(
    input clk,
    output logic out
);

logic [23:0] cnt;
logic [23:0] cnt__nxt;
logic out__nxt;

assign cnt__nxt = (cnt == MAX_COUNT) ? 0 : cnt + 1;
assign out__nxt = (cnt == MAX_COUNT) ? ~out : out;

always @(posedge clk)
begin
    cnt <= cnt__nxt;
end

always @(posedge clk)
begin
    out <= out__nxt;
end

endmodule

How It Works

To generate a 1 Hz square wave, the design uses a counter (cnt) that increments on each clock cycle. When the counter reaches MAX_COUNT, it resets to 0 and toggles the LED output.

For a 12 MHz clock (83.33 ns period), MAX_COUNT is set to 6,000,000 so that the LED toggles every 0.5 seconds (once per second for a complete on/off cycle). The counter is 24 bits wide to accommodate this value.

Note that this simple design doesn't include a reset signal. Instead, it relies on the FPGA's flip-flops being automatically initialized to 1'b0 during configuration. This choice was made so the design works without having to connect an external button or switch to drive a reset pin.

Pin Constraints

Next, we need to map the design's ports to physical pins on the FPGA using a pin constraints file.

Pin constraints files are generally simple. Each line in the file maps a signal in the design to a phsical pin on the FPGA package. The basic fomat is shown below.

Terminal
set_io <signal_name> <pin_number>

For the blinky project, create a file called blinky.pcf and add the following contents:

Terminal
set_io clk C4
set_io out E1

The first assignment connects clk to pin C4, which is connected to the 12 MHz oscillator on the Bluejay board. The second assignment connects out to pin E1, which drives the USR LED via header pin D16. See the pin mapping table for the complete pinout.

Generating the Bitstream

With our design files created, we can now generate the bitstream. This is done in a three-step process: logic synthesis, place-and-route, and bitstream packing.

To keep the workspace organized, we will store all generated files in a gen/ subdirectory. This keeps the design files separate from the outputs created by the bitstream tools. Before moving on, create the subdirectory gen/ by running the command:

Terminal
mkdir gen/

Synthesis

The first step in the bitstream generation process is to run logic synthesis, which converts the high-level RTL into a netlist of FPGA primitives.

For synthesis, we will be using Yosys. Yosys reads in a set of Verilog/SystemVerilog files and outputs a JSON file containing the synthesized netlist. It can be run with the following command:

Terminal
yosys -p "synth_ice40 -top blinky -json gen/blinky.json" blinky.sv

In the command above, the -top blinky option specifies that the top module in the design hierarchy is named blinky. The -json gen/blinky.json option specifies that the netlist should be written into a JSON file named gen/blinky.json. The command ends with a list of Verilog/SystemVerilog files containing the design's RTL. In our case, this list consists of a single file, blinky.sv.

Place and Route

After synthesis, the next step is to run place-and-route, which maps the netlist to physical resources on the FPGA.

For place-and-route, we will be using NextPNR. NextPNR reads in the synthesized netlist (generated in the previous step) and our pin constraints file and outputs the placed-and-routed design in an ASCII file. It can be run with the following command:

Terminal
nextpnr-ice40 --lp8k --package cm81 --json gen/blinky.json --pcf blinky.pcf --asc gen/blinky.asc

The --lp8k and --package cm81 options specify the target FPGA, which in our case is the iCE40LP8K-CM81. The output of the tool will be saved to the file gen/blinky.asc.

Bitstream Packing

The final step is to convert the placed design into a binary bitstream that can be loaded onto the FPGA.

For this step, we will use the IcePack tool (from Project IceStorm). IcePack reads in the placed-and-routed design and outputs a bitstream for the iCE40LP8K. It can be run with the following command:

Terminal
icepack gen/blinky.asc gen/blinky.bin

The bitstream generated by the tool will be saved to gen/blinky.bin. This file contains 1,081,464 bits of configuration data for the iCE40LP8K. For more details on the bitstream format, refer to the "iCE40 Programming and Configuration" manual.

Programming the FPGA

With the bitstream generated, it's time to load the design onto the FPGA. To do this, we will be using IceProg and an FTDI-based programmer.

Bitstreams are loaded onto the iCE40 using a SPI interface (CS, SCK, MOSI, MISO) and two dedicated configuration signals, CRESET_B (abbreviated CRST) and CDONE (abbreviated CDN). The Bluejay board's CONFIG interface provides access to these signals (as well as power and ground) for programming the device.

An FTDI-based programmer is needed to connect a computer to the board's CONFIG interface. FTDI chips, such as the FT232H, are commonly used to perform USB-to-serial protocol conversion. They allow USB-capable devices (such as a laptop or PC) to communicate with digital circuitry using simple serial interfaces, such as SPI, I2C, and UART. In our case, we will be using an FTDI-based programmer to perform USB-to-SPI conversion.

The IceProg tool is a simple, FTDI-based driver program designed to load bitstreams onto iCE40 FPGAs.

Two common programmer options are described below: the FTDI C232HM USB-to-serial cable and the Adafruit FT232H breakout board. The following sections detail the setup for each option.

The FTDI C232HM cable has a USB Type-A connector on one end and 10 color-coded wires with female headers on the other. The table below describes the pinout of the connector:

Adafruit FT232H Breakout Board

The Adafruit FT232H breakout board connects to a PC or laptop via a USB-C cable. The I/O of the FT232H chip are broken out on two 11-pin connectors. This includes both the SPI signals and several GPIO signals (which we will connect to CRST and CDN).

Board Setup

To load a bitstream directly onto the FPGA, the configuration mode jumper should be placed in the SM (Slave Mode) position as shown below. To enable the LEDs, the LED EN jumper should be connected.

Note that the USB and 5V pins should be disconnected. We will be supplying power to the board using the programmer through the 3V3 and GND pins.

Programming Command

Once the programmer has been connected and the configuration mode jumper is set to the SM position, run the following command to load the bitstream onto the FPGA:

sudo iceprog -d i:0x0403:0x6014 -S gen/blinky.bin

The -S flag tells the tool to program the FPGA's SRAM directly. If successful, the USR LED should start blinking at 1 Hz.

Programming the Flash (Optional)

To make your design persistent across power cycles, you can program the onboard flash chip. The FPGA will then automatically configure itself from the flash on each power-up.

Board Setup

To program the flash, the board's configuration mode jumpers should be set to the MM (Master Mode) position. Everything else, including the programmer connections, can remain the same. The setup is shown below.

Programming Command

To program the flash, run the following command:

sudo iceprog -d i:0x0403:0x6014 gen/blinky.bin

Note the absence of the -S flag. Without it, IceProg defaults to programming the flash chip rather than the FPGA SRAM.

After programming completes, IceProg releases the FTDI pins, allowing CS and CRST to be pulled high. This prompts the FPGA to enter Master Configuration Mode and load the bitstream from flash. The USR LED should start blinking immediately.

You can now disconnect the programmer. The design will automatically load whenever the board is powered on.

Next Steps

Congratulations! You've successfully created and programmed your first FPGA design on the Bluejay.
