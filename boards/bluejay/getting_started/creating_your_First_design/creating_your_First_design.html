Creating Your First Design

Overview

In this section, you'll create a simple LED blinker design. This design will blink the Bluejay's USR LED on and off at 1 Hz, demonstrating the basics of writing RTL code and mapping signals to physical pins.

The Blinky Module

Create a file called blinky.sv with the following SystemVerilog code:

Systemverilog
module blinky
#(parameter MAX_COUNT = 6000000)
(
    input clk,
    output logic out
);

logic [23:0] cnt;
logic [23:0] cnt__nxt;
logic out__nxt;

assign cnt__nxt = (cnt == MAX_COUNT) ? 0 : cnt + 1;
assign out__nxt = (cnt == MAX_COUNT) ? ~out : out;

always @(posedge clk)
begin
    cnt <= cnt__nxt;
end

always @(posedge clk)
begin
    out <= out__nxt;
end

endmodule

The module has one input, clk, which connects to the board's 12 MHz oscillator, and one output, out, which connects to the USR LED.

How It Works

To generate a 1 Hz square wave, the design uses a counter (cnt) that increments on each clock cycle. When the counter reaches MAX_COUNT, it resets to 0 and toggles the LED output.

For a 12 MHz clock (83.33 ns period), MAX_COUNT is set to 6,000,000 so that the LED toggles every 0.5 seconds (once per second for a complete on/off cycle). The counter is 24 bits wide to accommodate this value.

Note that this simple design doesn't include a reset signal. Instead, it relies on the FPGA's flip-flops being automatically initialized to 1'b0 during configuration. This choice was made so the design works without having to connect an external button or switch to drive a reset pin.

Pin Constraints

Next, we need to map the design's ports to physical pins on the FPGA using a pin constraints file.

Pin constraints files are simple - each line maps a signal in the design to a physical pin on the FPGA package. The basic format is:

set_io <signal_name> <pin_number>

Create a file called blinky.pcf with the following contents:

set_io clk C4
set_io out E1

The first assignment connects clk to pin C4, which is connected to the 12 MHz oscillator on the Bluejay board. The second assignment connects out to pin E1, which drives the USR LED via header pin D16.

For the complete board pinout, see the Board I/O page.

Project Structure

At this point, your project directory should contain two files:

blinky.sv       # The RTL design
blinky.pcf      # The pin constraints

Next Steps

With your design files created, you're ready to build the bitstream!
