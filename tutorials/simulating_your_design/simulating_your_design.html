Simulating Your Design

Overview

Before programming your design onto the FPGA, it's helpful to simulate it to verify correct behavior. This tutorial demonstrates how to use Icarus Verilog to simulate a design and GTKWave to view the resulting waveforms.

We'll use the simple blinky module from the Getting Started tutorial as our design under test (DUT).

What You'll Need

You'll need the following tools installed:

  • Icarus Verilog (iverilog) - Verilog simulator
  • GTKWave - Waveform viewer

Both tools are included in the OSS CAD Suite from the Getting Started tutorial.

Design Files

For this tutorial, we'll work with two files: the design module and a testbench.

blinky.sv

This is the same blinky module from the Getting Started tutorial:

module blinky
#(parameter MAX_COUNT = 6000000)
(
    input clk,
    input rst,
    output logic out
);

logic [23:0] cnt;
logic [23:0] cnt__nxt;
logic out__nxt;

assign cnt__nxt = (cnt == MAX_COUNT) ? 0 : cnt + 1;
assign out__nxt = (cnt == MAX_COUNT) ? ~out : out;

always @(posedge clk)
begin
    cnt <= rst ? 0 : cnt__nxt;
end

always @(posedge clk)
begin
    out <= rst ? 0 : out__nxt;
end

endmodule

tb_blinky.sv

The testbench instantiates the blinky module and provides stimulus:

`timescale 1ns/1ps

module tb_blinky;

logic clk;
logic rst;
logic out;

blinky #(.MAX_COUNT(2)) dut (
    .clk(clk),
    .rst(rst),
    .out(out)
);

initial
begin
    $dumpfile("tb_blinky.vcd");
    $dumpvars(0, tb_blinky);
end

initial
begin
    clk = 0;
end

always
begin
    #5 clk = ~clk;
end

initial
begin
    rst = 1;
    #10;
    rst = 0;

    #1000; 
    $finish;
end

endmodule

How the Testbench Works

The testbench sets up the simulation environment:

  1. Timescale: The `timescale 1ns/1ps directive sets the time unit to 1 nanosecond with 1 picosecond precision

  2. Clock generation: The always block toggles clk every 5 ns, creating a 100 MHz clock (10 ns period)

  3. Reset sequence: The reset signal starts high, then goes low after 10 ns to initialize the design

  4. Parameter override: MAX_COUNT is set to 2 (instead of 6,000,000) to make the simulation run faster. This means the output toggles every 3 clock cycles instead of every 6 million.

  5. Waveform dumping: The $dumpfile and $dumpvars system tasks create a VCD (Value Change Dump) file that records all signal transitions

  6. Simulation end: The $finish task stops the simulation after 1000 ns

Running the Simulation

You can run the simulation manually or use a Makefile to automate the process.

Manual Simulation

To compile and run the simulation manually:

# Create output directory
mkdir -p gen

# Compile the design and testbench
iverilog -g2012 -o gen/tb_blinky.vvp tb_blinky.sv blinky.sv

# Run the simulation
cd gen && vvp tb_blinky.vvp

This generates a gen/tb_blinky.vcd file containing the waveform data.

Using a Makefile

To streamline the simulation workflow, you can use the following Makefile:

# Simulation configuration
TB_MODULE = tb_blinky
TB_SOURCES = tb_blinky.sv blinky.sv
SIM_DIR = gen

# Tool configuration
IVERILOG = iverilog
VVP = vvp
GTKWAVE = gtkwave

# Simulation targets
VVP_FILE = $(SIM_DIR)/$(TB_MODULE).vvp
VCD_FILE = $(SIM_DIR)/$(TB_MODULE).vcd

# Create simulation directory
$(SIM_DIR):
    mkdir -p $(SIM_DIR)

# Compile simulation
$(VVP_FILE): $(TB_SOURCES) | $(SIM_DIR)
    $(IVERILOG) -g2012 -o $@ $(TB_SOURCES)

# Run simulation
$(VCD_FILE): $(VVP_FILE)
    cd $(SIM_DIR) && $(VVP) $(notdir $(VVP_FILE))

# Simulation target
sim: $(VCD_FILE)
    @echo "Simulation complete. Waveform saved to $(VCD_FILE)"

# Open waveforms in GTKWave
waves: $(VCD_FILE)
    $(GTKWAVE) $(VCD_FILE)

# Clean simulation artifacts
clean:
    rm -rf $(SIM_DIR)

# Phony targets
.PHONY: sim waves clean

With the Makefile in place, you can run:

# Run simulation
make sim

# View waveforms
make waves

# Clean up
make clean

Viewing Waveforms with GTKWave

After running the simulation, open the waveform viewer:

gtkwave gen/tb_blinky.vcd

Or if using the Makefile:

make waves

Navigating GTKWave

When GTKWave opens:

  1. Select signals: In the left panel under "SST" (Signal Selection Tree), expand tb_blinky to see all signals
  2. Add to waveform: Select signals (clk, rst, out, dut.cnt) and click "Append" or press Insert
  3. Zoom to fit: Click the "Zoom Fit" button or press Shift+Alt+F to see the entire simulation
  4. Zoom in/out: Use the + and - keys or scroll wheel to adjust the time scale
  5. Measure time: Click and drag to measure time intervals between events

What to Look For

In the waveform, you should observe:

  • Reset behavior: rst starts high, then goes low at 10 ns
  • Counter operation: cnt increments from 0→1→2, then resets to 0
  • Output toggling: out toggles each time cnt reaches MAX_COUNT (2)
  • Clock relationship: All transitions occur on rising edges of clk

Modifying the Simulation

To run longer simulations or test different scenarios, modify the testbench:

Longer simulation time:

#10000;  // Run for 10,000 ns instead of 1,000 ns
$finish;

Different MAX_COUNT value:

blinky #(.MAX_COUNT(5)) dut (  // Toggle every 6 cycles instead of 3
    .clk(clk),
    .rst(rst),
    .out(out)
);

Test multiple resets:

initial
begin
    rst = 1;
    #10;
    rst = 0;
    #200;
    rst = 1;  // Second reset
    #10;
    rst = 0;
    #1000;
    $finish;
end
